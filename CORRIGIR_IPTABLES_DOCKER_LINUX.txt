  CORRIGIR IPTABLES E REINICIAR DOCKER - LINUX
═══════════════════════════════════════════════════════════════

PROBLEMA: Chain 'DOCKER-ISOLATION-STAGE-2' does not exist
CAUSA: Regras iptables do Docker foram corrompidas
SOLUÇÃO: Recriar chains do Docker

═══════════════════════════════════════════════════════════════
  EXECUTAR NO LINUX (via PuTTY) - COPIAR E COLAR TUDO
═══════════════════════════════════════════════════════════════

cd /home/administrador/CorujaMonitor
sudo systemctl stop docker
sudo iptables -t nat -F
sudo iptables -t mangle -F
sudo iptables -F
sudo iptables -X
sudo systemctl start docker
sleep 10
sudo docker-compose up -d
docker ps
curl http://localhost:8000/health

═══════════════════════════════════════════════════════════════
  EXPLICAÇÃO DOS COMANDOS
═══════════════════════════════════════════════════════════════

1. systemctl stop docker
   → Para o Docker daemon

2. iptables -t nat -F
   → Limpa regras NAT

3. iptables -t mangle -F
   → Limpa regras MANGLE

4. iptables -F
   → Limpa regras FILTER

5. iptables -X
   → Remove chains customizadas (incluindo DOCKER-*)

6. systemctl start docker
   → Inicia Docker (recria chains automaticamente)

7. docker-compose up -d
   → Sobe containers
